## Lab 2 — Реалізація REST API для обліку витрат

### Загальний опис

У межах лабораторної роботи реалізовано REST API, яке дозволяє керувати трьома основними сутностями:

* **Users** — користувачі системи
* **Categories** — категорії витрат
* **Records** — записи про витрати користувачів

API підтримує створення, перегляд, фільтрацію та видалення даних.


## Реалізовані ендпоінти

###  Users

* `POST /user` — створити нового користувача
* `GET /users` — отримати список усіх користувачів
* `GET /user/<user_id>` — отримати інформацію про користувача за його ID
* `DELETE /user/<user_id>` — видалити користувача


###  Categories

* `POST /category` — створити категорію витрат
* `GET /category` — отримати всі категорії
* `DELETE /category/<category_id>` — видалити категорію


###  Records

* `POST /record` — створити запис витрати
* `GET /record/<record_id>` — отримати запис за ID
* `GET /record?user_id=X` — отримати записи вибраного користувача
* `GET /record?category_id=X` — отримати записи певної категорії
* `GET /record?user_id=X&category_id=Y` — фільтрувати записи одночасно за користувачем та категорією
* `DELETE /record/<record_id>` — видалити запис

Якщо запит `GET /record` виконується **без параметрів**, API повертає помилку — це відповідає вимогам лабораторної.


## Тестування через Postman

Для зручності тестування створено Postman-колекцію, що додається у каталозі `postman/`:

* `Backend_Labs_Expense_Tracker.postman_collection.json` — основна колекція запитів
* `Local.postman_environment.json` — конфігурація для локальної розробки
* `Production.postman_environment.json` — конфігурація для задеплоєного сервісу

Колекція включає всі ендпоінти та приклади використання.
